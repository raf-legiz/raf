# Настраиваем файл, для быстрой загрузки сервера

Если всё успешно, то теперь в Windows настроим файл конфигурации SSH, чтобы подключаться просто по имени сервера. Для этого создадим файл ".ssh\config"в директории пользователя (команда для powershell, можете создать этот файл и в ручную в папке вашего пользователя: "C:\Users\user"):

```shell
New-Item -Path $HOME\.ssh\config -ItemType File
```

Откроем его с помощью блокнота:

```shell
Notepad $HOME\.ssh\config
```
И заполним следующим содержимым (имя vps сервера может быть любым, указываете свой ip-адресс, имя пользователя и порт):

!!! example

    ```bash
    Host vps_server                 # (1)!
    HostName IP                     # (2)!
    StrictHostKeyChecking no        # (3)!
    User user_name                  # (4)!
    ForwardAgent yes                # (5)!
    IdentityFile ~/.ssh/id_ed25519  # (6)!
    IdentitiesOnly yes              # (7)!
    AddKeysToAgent yes              # (8)!
    ServerAliveInterval 60          # (9)!
    ServerAliveCountMax 1200        # (10)!
    Port 43921                      # (11)!
    ```

    1.  Вместо **vps_server** пишем своё название профиля.  
        Это алиас, по которому вы будете подключаться: `ssh vps_server`.

    2.  **IP** — прописываем реальный IP‑адрес или доменное имя сервера.

    3.  `StrictHostKeyChecking no` — отключает запрос подтверждения ключа сервера.  
        Удобно при тестах, но снижает безопасность. В проде лучше `ask` или `yes`.

    4.  **user_name** — указываем имя пользователя, которое сами создали на сервере  
        (НЕ root, если root‑логин отключён).

    5.  `ForwardAgent yes` — включает проброс SSH‑агента.  
        Позволяет использовать локальные ключи на удалённом сервере.

    6.  Указываем месторасположение приватного ключа.  
        Обычно файл находится в `~/.ssh/id_ed25519` или `id_rsa`.

    7.  `IdentitiesOnly yes` — заставляет SSH использовать только указанный ключ,  
        а не все ключи, загруженные в ssh-agent.

    8.  `AddKeysToAgent yes` — автоматически добавляет ключ в ssh-agent при подключении.

    9.  `ServerAliveInterval 60` — клиент отправляет keep‑alive пакет каждые 60 секунд,  
        чтобы соединение не разрывалось.

    10. `ServerAliveCountMax 1200` — количество keep‑alive пакетов до разрыва.  
        1200 × 60 = 20 часов стабильного соединения.

    11. `Port 43921` — порт SSH, который вы указали в конфигурации сервера (`sshd_config`).

Теперь подключиться можно простой командой:

```bash
ssh vps_server # (1)!
```

1. где `vps_server` — вводим своё название
